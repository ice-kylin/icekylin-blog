# 1.1 计算机安全的概念

# 1.1.1 计算机安全的定义

  • 对于一个自动化的信息系统，采取保护措施确保信息系统资源（包括硬件、软件、固件、信息 / 数据和通信）的完整性、可用性和保密性
    • 保密性
      • 数据保密性
      • 隐私性
    • 完整性
      • 数据完整性
      • 系统完整性
    • 可用性
  • 额外的安全概念
    • 真实性
    • 可追溯性

# 1.2 OSI 安全架构

  • 安全攻击：任何危及信息系统安全的行为
  • 安全机制：用来检测、阻止攻击或从攻击状态恢复到正常状态的过程（或实现该过程的设备）
  • 安全服务：加强数据处理系统和信息传输的安全性的一种处理过程或通信服务，目的在于利用一种或多种安全机制进行反攻击

# 1.3 安全攻击

# 1.3.1 被动攻击

  • 被动攻击的特性是对传输进行窃听和监测
    • 对于被动攻击，重点是预防而非监测

# 1.3.2 主动攻击

  • 主动攻击包括对数据流进行修改或伪造数据流，具体分为
    • 伪装
    • 重放
    • 消息修改
    • 拒绝服务
  • 主动攻击难以绝对预防，对于主动攻击，重点在于检测并从攻击造成的破环或延迟中恢复过来

# 1.4 安全服务

  • 在通信开放系统中，为系统或数据传输提供足够安全的协议层服务
  • 安全服务是一种由系统提供的对系统资源进行特殊保护的处理或通信服务；安全服务通过安全机制来实现安全策略
  • 分为 5 类 14 个特定服务
    • 认证
      • 同等实体认证
      • 数据源认证
    • 访问控制
    • 数据保密性
      • 连接保密性
      • 无连接保密性
      • 选择域保密性
      • 流量保密性
    • 数据完整性
      • 具有恢复功能的连接完整性
      • 无恢复的连接完整性
      • 选择域连接完整性
      • 无连接完整性
      • 选择域无连接完整性
    • 不可否认性
      • 源不可否认性
      • 宿不可否认性

# 1.7 攻击面与攻击树

# 1.7.1 攻击面

  • 攻击面由系统中一系列可访问且可利用的漏洞组成
    • 网络攻击面
    • 软件攻击面
    • 人类攻击面

# 1.7.2 攻击树

  • 攻击树是采用分支化、层次化来表示利用安全漏洞的可能技术集合的数据结构
    • 根节点：目标（造成安全事件）
    • 子节点：子目标（与节点 / 或结点）
    • 叶子节点：发动攻击的不同方式

# 思考题

# 1.1 什么是 OSI 安全架构？

参考答案
  • OSI 安全架构是一种提供安全的组织方法
  • OSI 安全架构是一种概念性和功能性的框架，给出了一种系统化的定义方法
  • 主要对安全攻击、安全机制和安全服务给出了定义和综述

# 1.2 被动安全威胁与主动安全威胁有何区别？

参考答案

被动安全威胁与主动安全威胁都是因为系统的脆弱性被利用而可能带来的危险，区别在于：

  • 被动安全威胁可能带来的危险是攻击者
    • 获取或利用系统的信息
    • 但并不影响系统的资源
  • 主动安全威胁可能带来的危险是攻击者
    • 改变系统资源或影响系统运行

# 1.3 列出并简要定义被动安全攻击与主动安全攻击的种类

参考答案
  • 被动安全攻击
    • 窃听（信息内容泄露）：攻击者获取传输的敏感或秘密信息
    • 监测（流量分析）：攻击者获取传输的敏感或秘密信息的一些模式并判断通信的某些性质
  • 主动安全攻击
    • 伪装：某实体假装成其它实体
    • 重放：攻击者未经授权地将截获的消息再次发送
    • 消息修改：攻击者未经授权地
      • 修改合法消息的一部分
      • 延迟消息的传输
      • 改变消息的顺序
    • 拒绝服务：阻止或禁止对通信设施的正常使用或管理

# 1.4 列出并简要定义安全服务的种类

参考答案
  • 认证：保证通信实体是它所声称的实体，包括同等实体认证和数据源认证
  • 访问控制：阻止对资源的非授权使用
  • 数据保密性：保护数据免于非授权泄露
    • 可应用于消息流、单条消息或消息的指定部分
    • 此外还包括保护可以通过观察流量而获得的信息（流量保密性）
  • 数据完整性：保证收到的数据是授权实体发出的数据
    • 可应用于消息流、单条消息或消息的指定部分
    • 此外还分为可恢复和无恢复的服务
  • 不可否认性：防止整个或部分通信过程中，任一通信实体（源 / 宿）进行否认的行为
  • 可用性服务：确保系统的可用性

# 1.5 列出并简要定义安全机制的种类

参考答案
  • 特定的安全机制：可以并入适当的协议层以提供一些 OSI 安全服务
  • 普遍的安全机制：不局限于任何特定 OSI 安全服务或协议层的机制

# 1.6 列出并简要定义基本的安全设计准则

参考答案
  • 机制的经济性：嵌入在硬件和软件中的安全设计应设计得尽量简单、短小
  • 故障安全默认：访问决策应基于允许而非拒绝
  • 完整的监察：必须检查访问控制机制中的每个访问
  • 开放的设计：安全机制的设计应该是开放的，而不是保密的
  • 权限分离：需要多个权限属性来访问一个受限资源
  • 最小权限：系统的每个进程和用户应该以执行该任务所需的最小权限集来进行操作
  • 最小共同机制：把不同用户共享的功能实现得最小化
  • 心理接受度：安全机制在满足授权访问的用户需求的同时，不应过度干预用户的工作
  • 隔离
    1. 公共访问系统应与关键资源（数据、进程等）隔离
    2. 各个用户的进程和文件因彼此隔离
    3. 安全机制也应被隔离
  • 密封：基于面向对象的特定形式的隔离
  • 模块化
    • 将安全功能作为单独的模块开发
    • 用于机制设计和实现的模块化架构
  • 分层：使用多个叠加的保护方法来保护信息系统的人员、技术和操作
  • 最小意外：程序或用户接口应让用户感到，对意外事故的处理响应最小化

# 1.7 解释攻击面与攻击树的区别

参考答案
  • 攻击面由系统中一系列可访问且可利用的漏洞组成
  • 而攻击树是采用分支化、层次化来表示利用安全漏洞的可能技术集合的数据结构
