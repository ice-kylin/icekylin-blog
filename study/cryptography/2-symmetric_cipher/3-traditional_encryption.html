# 📗 重点笔记

  • 明文：原始的消息
  • 密文：加密后的消息
  • 加密：明文到密文的变换过程
  • 解密：密文到明文的变换过程

# 3.1 对称密码模型

  • 对称加密方案的五个基本成分
    • 明文
    • 加密算法：加密算法对明文进行各种
    • 密钥
      • 密钥也是加密算法的输入
      • 密钥独立于明文和算法
      • 算法根据所用的特定密钥而产生不同的输出
      • 算法所用的确切替代和变换也依靠密钥
    • 密文
    • 解密算法
      • 本质上是加密算法的逆运算
      • 输入密文和密钥，输出原始密文

# 3.1.1 密码编码学

  • 密码编码学系统具有以下三个独立的特征
    • 转换明文为密文的运算类型
      • 替代：将明文中的每个元素（如位、字母、位组或字母组等）映射成另一元素
      • 置换：将明文中的元素重新排列
    • 所用的密钥数
      • 对称密码 / 单密钥密码 / 秘密钥密码 / 传统密码：发送方和接受方使用相同的密钥
      • 非对称密码 / 双钥 / 公钥密码：发收双方使用不同的密钥
    • 处理明文的方法
      • 分组密码：每次处理输入的一组元素，相应的输出一组元素
      • 流密码：连续地处理输入元素，每次输出一个元素

# 3.1.2 密码分析学和穷举攻击

  • 密码分析学
    • 依赖于算法的性质、明文的一般特征或某些明密文对
    • 企图利用算法的特征来推到出特定的明文或使用的密钥
  • 穷举攻击
    • 攻击者对一条密文尝试所有可能的密钥直到把它转换为可读的有意义的明文
    • 除非明文已知，否则分析者需要能够将明文识别为明文
    • 平均而言，获得成功至少要尝试所有可能密钥的一半
攻击类型密码分析者已知的信息
唯密文攻击加密算法、密文
已知明文攻击加密算法、密文、用同一密钥加密的一个或多个明密文对
选择明文攻击加密算法、密文、分析者选择的明文及对应的密文
选择密文攻击加密算法、密文、分析者选择的一些密文及对应的明文
选择文本攻击加密算法、密文、分析者选择的明文及对应的密文、分析者选择的一些密文及对应的明文

💡 tip

已知明文攻击和选择明文攻击的相同点和区别：

  • 相同点
    • 都已知加密算法、密文
  • 区别
    • 已知明文攻击和可能词攻击紧密相关
      • 攻击者可以从明文信息的一些固定格式或关键词入手推导出密钥
    • 已知明文攻击无法获得信源系统以让发送方在发送时插入一段由它选择的信息

# 🪶 已知明文攻击的例子

二战期间，英国以图灵为代表的团队破译德国的恩尼格玛机过程中，发现德国明文中经常出现一些关键词 —— 如无特殊情况、希特勒万岁等；又如早上 6 点钟德国人喜欢发送一条天气预报，明文开头中肯定包含天气这个词，使得图灵团队极大地加快了破译密钥的时间

  • 一般地，加密算法起码要禁受得住已知明文攻击
  • 如果一个密码体制满足条件：无论有多少可使用的密文，都不足以唯一地确定密文所对应的明文，则称该加密体制是无条件安全的
    • 除了一次一密，所有的加密算法都不是无条件安全的
  • 如果满足以下两条标准的任意一条，则它是计算上安全的
    • 破译密码的代价超出密文信息的价值
    • 破译密码的时间超出密文信息的有效生命期
  • 对称密码体制的所有分析方法都利用了一个事实：明文的结构和模式在加密之后仍然保存了下来，并能在密文中找到一些蛛丝马迹

# 3.2 代替技术

# 3.2.1 Caesar 密码

# 🪶 凯撒密码的历史

该密码是以罗马共和时期恺撒的名字命名的，当年恺撒曾用此方法与其将军们进行联系

如果需要保密，信中便用暗号，也即是改变字母顺序，使局外人无法组成一个单词。如果想要读懂和理解它们的意思，得用第 4 个字母置换第一个字母，即以 D 代 A，余此类推。

—— 苏维托尼乌斯，罗马十二帝王传

  • 对字母表中的每个字母，用它之后的第 3 个字母来代替
  • 让每个字母等价于一个数值
    • abcdefghijklm
      0123456789101112
      nopqrstuvwxyz
      13141516171819202122232425
  • 加密算法可以表达如下，对每个明文字母 p，代替成密文字母 C
    • C = E(3, p) = (p + 3) mod (26)
  • 移位可以是任意大于 1 小于 25 的整数 k，这样就得到了一般的 Caesar 算法
    • C = E(k, p) = (p + k) mod (26)
  • 解密算法是
    • p = D(k, C) = (C - k) mod (26)
-- 26 English letters only
ce :: Int -> String -> String
ce k xs = [i2c (caesarE k (c2i x)) | x <- xs]

cd :: Int -> String -> String
cd k xs = [i2c (caesarD k (c2i x)) | x <- xs]

caesarE :: Int -> Int -> Int
caesarE k p = (p + k) `mod` 26

caesarD :: Int -> Int -> Int
caesarD k = caesarE (- k)

alphabet :: String
alphabet = "abcdefghijklmnopqrstuvwxyz"

i2c :: Int -> Char
i2c i = alphabet !! i

c2i :: Char -> Int
c2i c = index c alphabet

index :: (Eq t) => t -> [t] -> Int
index _ [] = -1
index x (y : ys)
  | x == y = 0
  | otherwise = 1 + index x ys
  • Caesar 密码的三个重要特征使我们可以使用穷举攻击分析方法
    1. 已知加密和解密算法
    2. 需测试的密钥只有 25 个
    3. 密文所用的语言是已知的，且其意义易于识别
  • 一般来说，密钥空间很大的算法使得穷举攻击分析方法不太可能

💡 tip

密钥空间是指加密密钥大小的范围。

# 3.2.2 单表代替密码

  • 有限元素 S 的置换是 S 的所有元素的有序排列，且每个元素只出现一次
    • 如果 S = {a, b, c}，则 S 有 6 个置换
      • a, b, c
      • a, c, b
      • b, a, c
      • b, c, a
      • c, a, b
      • c, b, a
    • 一般地，具有 n 个元素的集合有 n! 个置换
  • 单表代替：在加密时用一张自制字母表上的字母来代替明文上的字母来加密

💡 tip

移位密码（如凯撒密码）也属于单表代替。

  • 分析方法
    1. 首先把字母的相对使用频率统计出来，与英文字母的使用频率分布进行比较
    2. 继续分析
      • 尝试做一些代替，填入明文，看看是否像一个消息的轮廓
      • 或者更系统的方式是寻找其它的规律
        • 明文中有某些词可能是已知的（可能词攻击）
        • 寻找密文字母中的重复序列，推导它们的等价明文（如统计双字母组合的频率）
    3. 继续进行类似的分析、测试
  • 对策
    • “同音词”方法：每个字母提供多种代替
    • 对明文中的多个字母一起加密
    • 采用多表代替密码

# 3.2.3 Playfair 密码

# 🪶 普莱费尔密码的历史

  • 该密码由英国科学家查尔斯·惠斯通（Charles Wheatstone）于 1854 年发明
  • 经莱昂·普莱费尔（Lyon Playfair）提倡在英国军队和政府使用
  • 在 1854 到 1855 年的克里米亚战争和 1899 年的布尔战争中有广泛应用
  • 在 1915 年的一战中被破译（一战中英军使用其作为陆军的战时加密体制）
  • 在二战中美军以及其它的一些盟友任在大量使用
  • Playfair 密码是最著名的多字母替代密码
  • 它将明文中的双音节字母作为一个单元并将其转换成密文的“双音节字母”
  • Playfair 算法基于一个由密钥词构成的 5 × 5 字母矩阵
    • 使用 monarchy 作为密钥词
    • 将密钥词（去掉重复字母）从左至右，从上至下填入矩阵格子中
    • 再将剩余字母的字母按字母表的顺序从左至右、从上至下填在矩阵剩下的格子里（I / J 暂时当成一个字母）
      • MONAR
        CHYBD
        EFGI / JK
        LPQST
        UVWXZ
    • 对明文按如下规则一次加密两个字母
      1. 如果该字母对的两个字母是相同的，那么在它们之间加一个填充字母（如 x）
        • ba ll oo n -> ba lx lo on
      2. 落在矩阵同一行的明文字母对中的字母
        • 由其右边的字母来代替
        • 每行中最右边的一个字母就用该列中最左边的一个字母来代替
          • ar -> RM
          • ch -> HY
      3. 落在矩阵同一列的明文字母对中的字母
        • 由其下面的字母来代替
        • 每行中最下面的一个字母就用该列中最上面的第一个字母来代替
      4. 其它的每组明文字母对中的字母按如下的方式代替
        • 该字母所在行为密文所在行
        • 该字母所在列为密文所在列
          • hs -> BP
          • ea -> IM / JM
  • Playfair 密码仍然是相对容易攻破的，因为它的密文仍然完好地保留了明文语言的大部分结构特征
